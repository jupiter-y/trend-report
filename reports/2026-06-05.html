AI가 AI를 만든다 — Anthropic 프로덕션 코드의 80%를 Claude가 작성

Anthropic의 전체 코드베이스에 병합된 코드 중 80% 이상이 이제 Claude에 의해 작성되며, 엔지니어 1인당 분기별 코드 출력량은 2024년 대비 8배에 달한다. 특히 명확한 사양이 없는 복잡한 오픈엔드 엔지니어링 문제에서 Claude의 성공률은 2026년 5월 기준 76%로, 불과 6개월 만에 50포인트 상승했다. 2026년 4월에는 Claude가 자율적으로 800개 이상의 개별 수정을 배포해 특정 API 오류율을 1,000배 낮췄으며, 담당 엔지니어 추산으로는 인간 개발자가 동일 작업에 4년이 필요했을 것으로 평가됐다. 엔터프라이즈 관점에서 이 전환은 단순 도구 도입을 넘어, "개발자 보조" 멘탈 모델을 완전히 버리고 "자동화 팩토리" 아키텍처로 이전할 것을 요구한다. 동시에 Anthropic의 Project Glasswing은 글로벌 디지털 인프라에서 수 주 만에 10,000개 이상의 고·치명적 취약점을 발견하며, 기업 사이버보안 과제를 '발견'에서 '패치 배포 속도'로 완전히 전환시켰다.

클라우드 없이도 멀티모달 — Google Gemma 4 12B, 16GB 노트북에서 완전 로컬 실행

많은 오픈소스 AI 모델 공급자들이 더 크고 강력한 모델을 추구하는 가운데, Google은 소형·로컬 시장에 집중해 Apache 2.0 라이선스의 오픈웨이트 모델 Gemma 4 12B를 16GB VRAM 표준 엔터프라이즈 노트북에서 완전 로컬 실행 가능하도록 최적화해 출시했다. 가장 주목할 혁신은 인코더 프리(Encoder-Free) "Unified" 아키텍처로, 원시 오디오 파형과 시각 패치를 별도 처리 모듈 없이 LLM 백본에 직접 입력해 지연 시간과 메모리 오버헤드를 대폭 절감했다는 점이다. Gemma 4 12B는 26B MoE 모델에 근접한 벤치마크 성능을 절반 이하의 메모리 풋프린트로 구현하며, 16GB RAM 노트북에서 강력한 멀티모달 및 에이전트 경험을 로컬에서 가능하게 한다. 헬스케어·금융·국방 등 규제 산업에서는 민감한 멀티모달 데이터를 온프레미스 또는 직원 노트북에서 처리할 수 있어 데이터 유출 위험을 제거하고 엄격한 컴플라이언스 프레임워크를 충족할 수 있다.

에이전트가 만드는 데이터 사일로를 막아라 — Microsoft IQ & Rayfin의 거버넌스 전략

엔터프라이즈 AI의 과제는 이제 모델 성능이 아닌 '일관된 공유 데이터 컨텍스트'로 전환됐다. 개발자와 비즈니스 사용자가 자동화하려는 것을 오늘날 모델들은 이미 구현할 수 있지만, 병목은 컨텍스트다. 새 에이전트가 배포될 때마다 비즈니스 작동 방식, 데이터 위치, 적용 규칙을 처음부터 다시 학습해야 하며, 일관된 기반 없이는 에이전트 간 조율과 확장이 불가능하다. 이에 Microsoft는 Build 2026에서 기존 Fabric IQ를 확장해 조직 운영 방식(Work IQ), 기관 지식(Foundry IQ), 실시간 웹 신호(Web IQ)를 포함하는 통합 시스템 Microsoft IQ를 발표해 에이전트가 4개의 컨텍스트 소스를 단일 기반으로 활용할 수 있게 했다. 더불어 에이전트가 생성한 애플리케이션 데이터가 Microsoft OneLake에 기본 착지하도록 하는 오픈소스 SDK·CLI인 Rayfin은, 아무런 거버넌스 없이 데이터 사일로를 양산하는 Supabase·Neon 방식에 대항해 전체 앱 플릿을 Fabric의 통합 거버넌스 레이어로 라우팅한다. 애널리스트들은 엔터프라이즈 AI의 다음 단계는 모델 레이어가 아니라, 거버넌스화된 기업 데이터를 안전한 운영 애플리케이션으로 전환할 수 있는 플랫폼이 승자가 될 것이라고 분석한다.

엔터프라이즈 소프트웨어 혁신: AI 에이전트가 소프트웨어 딜리버리 전 과정을 재편

Endava는 스스로를 '에이전틱 조직(Agentic Organization)'으로 선언하며, 시니어 엔지니어링 전문성을 AI 에이전트에 내재화해 전체 고객 참여 생애주기에 걸쳐 팀과 함께 협업하게 하는 방식을 도입했다. 에이전틱 워크플로우가 성숙한 팀에서는 개발자가 하루 업무 중 코딩에 쓰는 시간 비중이 60%에서 25%로 줄고, 아키텍처 설계 비중이 15%에서 55%로 급등하는 등 엔지니어 역할 자체가 근본적으로 재정의되고 있으며, 머지된 풀 리퀘스트는 98%나 증가한 것으로 보고된다. Endava는 OpenAI Codex를 단순 코드 자동완성이 아닌 요구사항 분석·설계·클라이언트 커뮤니케이션까지 아우르는 범용 데스크톱 에이전트로 활용하며, 실제로 2시간짜리 회의 녹취록 하나를 Codex에 투입해 수 주 걸리던 요구사항 명세서를 단 시간 만에 완성하는 성과를 거뒀다. 이는 한 기업의 변화를 넘어, 클라이언트가 AI 네이티브 딜리버리 모델을 요구함에 따라 전문 서비스 산업 전체가 진화해야 할 방향을 선도하는 사례로 평가된다.

오픈 LLM의 새 기준: NVIDIA Nemotron 3 Ultra, 에이전트용 550B 하이브리드 모델 공개

NVIDIA가 공개한 Nemotron 3 Ultra는 총 5,500억(활성 550억) 파라미터를 보유한 오픈 MoE(혼합 전문가) 모델로, 장시간 실행되며 계획 수립·도구 호출·다단계 추론을 수행하는 AI 에이전트를 주요 타깃으로 설계됐다. 핵심 아키텍처는 순수 트랜스포머 대신 Mamba 레이어와 Attention 레이어를 혼합한 하이브리드 구조를 채택해, 긴 시퀀스를 서브-제곱 비용으로 처리하면서도 대형 컨텍스트에서의 정밀 리콜 능력을 유지한다. 동급 오픈 LLM 대비 최대 약 6배 높은 추론 처리량을 달성하고, 100만 토큰 컨텍스트 윈도우를 지원하며 비환각 점수(AA-Omniscience) 78.7점으로 비교군 최고치를 기록했다. 가중치, 학습 데이터, 레시피 전체를 OpenMDW-1.1 라이선스 하에 완전 공개해 LoRA·SFT·RL을 통한 파인튜닝이 가능하며, Accenture, Palantir, Perplexity 등 13개 얼리어답터가 이미 채택을 확인한 상태다.

생성형 AI의 지속성 진화: ChatGPT 'Dreaming' 메모리로 대화를 넘어 개인 맥락을 기억하다

Dreaming은 ChatGPT가 백그라운드에서 과거 대화 이력을 자동으로 큐레이션하고 합성해, 항상 가장 최신의 관련 컨텍스트를 대화에 제공하는 방식이다. 이를 통해 "7월에 싱가포르 여행 예정"이라는 기억이 여행 종료 후 자동으로 "2026년 7월에 싱가포르를 다녀왔다"로 갱신되는 등 시간이 지남에 따라 메모리가 스스로 업데이트된다. 컴퓨팅 요구량을 기존 대비 약 5배 절감한 아키텍처 혁신 덕분에 OpenAI는 이 개인화 메모리 기능을 무료 사용자에게도 최초로 제공하고, 유료 Plus·Pro 사용자의 메모리 저장 용량은 2배로 확대했다. 한편 EU AI법상 2026년 8월부터 투명성 규정이 발효될 예정인 만큼, 사용자 행동을 추론·요약하는 메모리 시스템은 규제 당국이 집중 점검할 대상이 될 가능성이 높아 개인정보 보호 측면의 거버넌스 과제도 병존한다.

AI 비용 폭주 — 기업 예산이 몇 달 만에 소진되는 "토큰 쇼크" 시대

일부 기업들은 연간 AI 예산을 단 3개월 만에 소진했고, AI 지출이 두 배·세 배로 급증하는 사례도 속출하고 있다. 특히 Uber의 CTO는 2026년 전체 Claude Code 예산을 4개월 만에 써버렸으며, Uber COO는 높아진 사용량이 비례적으로 유용한 기능으로 이어지지 않는다고 지적했다. 이에 기업들은 가능한 한 많은 토큰맥시밍에서 벗어나 AI를 보다 효율적으로 사용하는 방향으로 선회하고 있다. 2026년 CMO들은 마케팅 예산의 평균 15.3%를 AI에 투자하지만, 이를 효과적으로 확장할 인프라·프로세스·역량을 갖춘 조직은 30%에 불과해 비용 대비 성과(ROI)를 입증하는 거버넌스 전략이 시급한 과제로 부상하고 있다. 올바른 모델을 올바른 업무에 매칭하는 능력, 즉 고비용 모델을 모든 작업에 남용하지 않는 기업이 비용 구조를 먼저 장악할 것이다.

Anthropic의 80% AI 코드 시대 — 엔지니어의 역할이 "작성자"에서 "감독자"로 전환

2026년 5월 기준, Anthropic 코드베이스에 병합된 코드의 80% 이상이 Claude에 의해 작성되었으며, 일반 엔지니어는 2024년 대비 8배 많은 코드를 분기별로 배포하고 있다. 개방형 복잡 문제에서 Claude의 성공률은 6개월 만에 76%로 50포인트 상승했으며, 내부 벤치마크에서 인간 개발자가 4~8시간을 들여 4배 속도 향상을 이루는 작업을 Claude는 52배 가속화했다. 실제로 2026년 4월, 한 Anthropic 엔지니어가 Claude를 배포해 800개 이상의 API 오류를 자율적으로 수정하고 에러율을 1,000배 감소시켰는데, 인간이라면 4년이 걸릴 작업이었다. 그러나 80% 자동화 코드베이스를 달성하려면 API 토큰 구매나 에이전트 루프 설정 이상으로, 개발자 역할의 전면 재정의와 자동화 검증 가드레일 구축이 필수적이다.

Microsoft IQ·Rayfin — AI 에이전트가 만들어내는 데이터 사일로 문제의 해법

AI 에이전트가 거버넌스보다 빠르게 애플리케이션을 생성하면서 각각이 데이터 레이어 외부에 새로운 사일로가 될 위험이 커지자, Microsoft가 Build 2026에서 이 문제를 정면 해결하겠다고 나섰다. Microsoft IQ는 Work IQ·Fabric IQ·Foundry IQ·Web IQ 등 4개 컨텍스트 소스를 단일 공유 기반으로 통합해, 에이전트가 일관된 추론과 기업 규모의 실행을 가능하게 한다. Rayfin은 에이전트가 구축한 애플리케이션을 Microsoft OneLake에 직접 배포해 데이터가 기본적으로 통합 플랫폼 안에 쌓이도록 하며, 컴플라이언스 레이어를 통해 격리된 사일로 생성을 원천 차단한다. 이 접근은 RAG와 모델 가용성이 더 이상 핵심 과제가 아니라는 업계 트렌드를 재확인시키며, 이제 진짜 문제는 Microsoft가 실행을 단순화하고 신뢰를 강화할 수 있는가라는 거버넌스 실행력의 문제다.
