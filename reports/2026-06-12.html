SkillOpt

Microsoft가 공개한 오픈소스 에이전트 스킬 자동 최적화 프레임워크. 딥러닝 스타일의 edit budget·validation gate·negative memory를 텍스트 문서에 적용한다.

edit budget

SkillOpt에서 한 스텝에 적용 가능한 최대 편집 횟수 제한. 딥러닝의 학습률(learning rate)과 동일한 역할로 스킬 문서의 급격한 변동을 방지한다.

validation gate

SkillOpt의 품질 관리 메커니즘. 후보 스킬이 held-out 검증 세트에서 실제로 성능을 높일 때만 채택되며, 그렇지 않으면 rejected-edit buffer로 전송된다.

negative memory

SkillOpt가 거부된 편집 내역을 기억해 동일한 실패 패턴이 반복되지 않도록 하는 부정 피드백 메커니즘.

LCLM (Latent Context Language Model)

NYU·Columbia 등 공동 연구팀이 개발한 인코더-디코더 구조의 컨텍스트 압축 모델. KV 캐시 방식과 달리 디코더 프리필 이전에 압축해 최대 16배 압축·8.8배 속도 향상을 달성한다.

KV 캐시 (Key-Value Cache)

트랜스포머 모델의 어텐션 레이어에서 키-값 쌍을 캐싱해 중복 연산을 줄이는 표준 추론 가속 기법. LCLM은 이 방식 대비 높은 압축률과 속도 향상을 보였다.

크로스세션 메모리 (Cross-Session Memory)

MiMo Code가 채택한 SQLite FTS5 기반 4계층 메모리 구조(프로젝트·세션·스크래치·태스크). 대화 세션이 종료되거나 컨텍스트 창이 초과해도 작업 상태를 유지한다.

checkpoint-writer

MiMo Code의 독립적인 서브에이전트. 주 코딩 에이전트가 작업에 집중하는 동안 병렬로 진행 상황을 기록해 컨텍스트 손실 없이 장기 작업을 이어갈 수 있게 한다.

MCP 서버 (Model Context Protocol Server)

AI 모델과 외부 도구·데이터 소스를 표준화된 방식으로 연결하는 프로토콜 서버. Grok Build, Hermes Agent 등 여러 에이전트 프레임워크에서 플러그인 연동에 활용된다.

commit-SHA

특정 Git 커밋을 고유하게 식별하는 40자 해시값. Grok Build 마켓플레이스는 모든 원격 플러그인을 commit-SHA로 고정해 포스 푸시나 리포지터리 침해를 통한 코드 주입을 방지한다.

프론티어 모델 (Frontier Model)

현재 기술 수준에서 가장 높은 성능을 보이는 최첨단 대규모 언어 모델군. GPT-5.5, Claude Opus 4.8, Gemini 3.1 Pro 등이 해당한다.

프레딕티브 하이어링 (Predictive Hiring)

과거 채용 데이터와 AI 분석을 활용해 후보자의 직무 적합성과 조직 성과를 예측하는 데이터 기반 채용 방식.

디지털 트윈 (Digital Twin)

크리에이터 이코노미 맥락에서 크리에이터 본인의 동의하에 구축된 AI 기반 자신의 복제 캐릭터. 무허가 음성 클론과 구별되는 합법적·윤리적 대안으로 논의된다.

음성 클론 (Voice Clone)

AI 기술로 특정 인물의 목소리를 복제하는 기법. 동의 없이 사용될 경우 저작권·프라이버시 침해 문제가 발생하며, 크리에이터 이코노미의 수익 구조를 위협한다.
